
# Best Practices: Как минимизировать галлюцинации AI-наставника

Для агента технической поддержки и онбординга критически важна точность: "галлюцинация" в команде CLI или параметре API может привести к ошибкам в проде. 

Ниже описаны методы защиты, специфичные для работы с документацией и техническими инструкциями.

---

## 1. Промпт-инжиниринг: Защита инструкций

Галлюцинации часто возникают при попытке "вспомнить" синтаксис редких библиотек.

### Negative Constraints (Явные запреты на выдумки)
Технические агенты любят "додумывать" аргументы функций. Запретите это на уровне промпта.

**Пример:**
```

system_prompt_additions = """
ЗАПРЕТЫ НА ВЫДУМЫВАНИЕ КОДА:

1. НЕ придумывай параметры команд CLI или аргументы функций, если их нет в найденной документации.
2. Если ты не уверен в синтаксисе, напиши: "Рекомендую проверить актуальный синтаксис в официальной документации" и дай ссылку.
3. НЕ смешивай инструкции для разных ОС (Windows/Linux) без явного разделения.
"""
```

### Role-Playing (Ролевая модель "Осторожный ментор")
Задайте роль не просто "эксперта", а "осторожного наставника", который перепроверяет себя.
> *Промпт:* "Ты — Senior DevOps инженер, обучающий стажера. Твоя цель — дать рабочий и безопасный пример. Ты предпочитаешь сказать 'я не знаю', чем дать команду, которая сломает прод."

## 2. Работа с источниками

Для технической документации качество поиска важнее креативности.

### Strict Domain Locking (Жесткий лок доменов)
В вашем коде уже реализован `SEARCH_DOMAINS = ['yandex.cloud']`. Это **фундаментальная защита**.
**Совет:** Добавьте в список разрешенных доменов `github.com/yandex-cloud` и `terraform.io` (если используются провайдеры), но явно запретите `stackoverflow.com` и `medium.com` для критических инструкций, так как там часто устаревшие решения.

### Version Awareness (Контроль версий)
Документация меняется. Агент может найти статью 2021 года.
**Решение:**
В системный промпт добавьте инструкцию приоритизации:
> "При противоречиях в документации отдавай предпочтение страницам с более свежей датой обновления или пометкой 'Current Version'. Игнорируй архивные разделы."

## 3. Pydantic-валидация для технических ответов

Вы уже используете `WebSearchAgentOutput`. Давайте усилим его валидаторами для проверки ссылок и безопасности.

### URL Hallucination Check
Агенты часто придумывают "красивые" ссылки, которые не существуют (404).

**Пример улучшения валидации:**
```python

from pydantic import BaseModel, validator, AnyHttpUrl

class WebSearchAgentSource(BaseModel):
url: AnyHttpUrl
title: str

    @validator('url')
    def validate_yandex_cloud_domain(cls, v):
        # Защита от ссылок на фишинговые или левые ресурсы, если агент решит их вставить
        allowed_hosts = ['yandex.cloud', 'cloud.yandex.ru']
        if v.host not in allowed_hosts:
            raise ValueError(f"Ссылка ведет на недоверенный домен: {v.host}")
        return v
```

### Code Sanitization (Очистка кода)
Если вы планируете выводить код, можно добавить пост-обработку, которая проверяет наличие опасных команд (например, `rm -rf /`, `sudo` без необходимости) в блоках кода Markdown перед показом пользователю.
