# Мультиагентный пайплайн подготовки коммерческого предложения
Мультиагентная система на базе **LangGraph** и **Yandex AI Studio**, которая по запросу менеджера автоматически готовит коммерческое предложение: подбирает сервисы и модели через *RAG* по документации, уточняет требования, рассчитывает стоимость и генерирует готовое КП.

---

## Содержание

### 1. Архитектура решения
*   Схема потоков данных: `Запрос менеджера` → `RAG-агент + Агент требований` → `Оркестратор` → `КП`.
*   Роли и права доступа сервисных аккаунтов (IAM).

### 2. Подготовка окружения
*   Установка библиотек (`openai`, `langgraph`, `curl_cffi`, `beautifulsoup4`, `pydantic`).
*   Инициализация клиента OpenAI-compatible API.

### 3. Парсинг документации и создание Vector Store
*   Парсинг страниц `aistudio.yandex.ru/docs` через `curl_cffi` (обход SmartCaptcha).
*   Подготовка и очистка данных для RAG.
*   Чанкование: автоматическое vs пользовательское (JSONL).
*   Загрузка чанков и создание Vector Store.
*   Проверка и оценка качества RAG.

### 4. RAG-агент: подбор сервисов и оценка стоимости
*   **File Search Tool** по Vector Store с документацией.
*   Подбор моделей, инструментов и оценка стоимости.

### 5. Агент уточнения требований
*   Формирование ТЗ на основе запроса менеджера.
*   Уточняющие вопросы и структурирование требований.

### 6. Оркестрация через LangGraph
*   Определение состояния графа (`TypedDict`).
*   Узлы графа: `rag`, `requirements`, `proposal`.
*   Параллельное выполнение `rag` и `requirements`.
*   Визуализация графа.

### 7. Запуск полного пайплайна
*   От запроса менеджера до готового КП в одном вызове.

### 8. Визуализация коммерческого предложения
*   Отображение КП в Markdown.
*   Детали работы каждого агента.

### 9. Очистка ресурсов
*   Удаление Vector Store и загруженных файлов.

## Best Practices

*   **RAG-качество**: Оценивайте качество поиска по Vector Store перед продакшеном — собирайте датасет вопросов и измеряйте метрики.
*   **Чанкование**: Используйте пользовательское чанкование (JSONL) когда нужен контроль над границами чанков, автоматическое — для быстрого старта.
*   **Оркестрация**: Запускайте независимых агентов параллельно через LangGraph для ускорения пайплайна.
*   **Промпт-инжиниринг**: Создавайте детальные системные промпты для каждого агента с описанием роли, формата и ограничений.
*   **Структурированный вывод**: Используйте Pydantic-модели для предсказуемого JSON от модели.
*   **Безопасность**: Храните API-ключи в `.env` файле, никогда не коммитьте их в репозиторий.
